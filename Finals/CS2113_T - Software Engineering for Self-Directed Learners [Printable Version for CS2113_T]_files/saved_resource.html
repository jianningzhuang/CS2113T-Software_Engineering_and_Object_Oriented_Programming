  • Tom

  • Suppose Tom has create a software on his Computer. Revision history of his code is in a local repo.

  • He now replicate it as a remote repo on GitHub and made is publicly available.

  • Jane notices the public repo and she wants to contribute to it.

  • She forks the repo onto her own GitHub account …

  • … which gives her a remote repo that is a copy of the Tom’s remote repo.

  • She clones the fork onto her Computer …

  • which gives her a local repo.

  • Now she has added a new feature, consisting of two commits. She wants to contribute the new feature to the main product.

  • First, she pushes the new commits to her fork ...

  • … which copies the new commits to her fork.

  • She creates a pull request to let Tom know her intention to contribute the new feature. GitHub notifies Tom that there is a new pull request.

  • Tom reviews the new code, agrees it is a good addition, and merges it to his remote repo …

  • … which copies the new commits to Tom’s remote repo.

  • Tom pulls the new commits to his local repo to sync his local repo with the remote repo.

  • Tom notices a bug in the new feature and fixes it, creating a new commit.

  • He pushes the new commit to his remote repo.

  • Jane notices the bug fix. She pulls the new commit directly from Tom’s remote repo to her local repo.
